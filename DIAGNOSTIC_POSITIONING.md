## Strategic Thesis

**AiVIS is not an AI optimization tool.**

AiVIS is the **post-mortem instrumentation layer** for AI visibility.

Everyone else teaches you _how to optimize_.  
We teach you _why the model decided against you_.

## Content Compiler Output Adjustment

### Pipeline Changes

**Before:** Entity extraction → Optimization suggestions → Page generation  
**After:** Entity extraction → **Diagnostic analysis** → Evidence forensics → Page generation

### New Output Artifacts

Instead of:

- "Recommendations"
- "Best practices"
- "Optimization checklist"

Generate:

- **Collapse Point Diagnosis** — Why citations fail for this entity
- **Decay Analysis** — Historical visibility loss patterns
- **Semantic Bridge Report** — Missing connections preventing retrieval
- **Competitor Map** — Why competitors rank for this entity
- **Recovery Evidence** — What signal patterns restore visibility
- **Prompt Routing Analysis** — Which search patterns miss you

### Ledger-Traceable Diagnostics

Every diagnostic output must:

- Reference specific `scan_id` + evidence events
- Connect to citation ledger entries (or explain absence)
- Show forensic drill-down (not just scores)
- Include comparative reasoning (not standalone claims)
